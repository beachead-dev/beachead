# Shared Memory

## Coming in Phase 3

Shared Memory allows multiple personas to access a common knowledge base
via a shared MCP server instance. This feature is planned for Phase 3 of
the Beachead orchestrator.

## Planned Capabilities

- Create shared memory instances with dedicated MCP containers
- Assign multiple personas to a shared memory instance
- Per-persona bearer tokens for access control
- Revoke access by removing a persona from the shared instance
- Shared memory data persisted via Docker volumes

## Current Status

This feature is not yet implemented. Per-persona memory (Phase 2) is
available now — enable it via the **Enable Memory** toggle on the persona
form.

For updates on shared memory development, check the project backlog.
